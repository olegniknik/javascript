# Проверка безопасности проекта «Уютный дом»

| # | Пункт | Статус | Комментарий |
|---|--------|--------|-------------|
| 1 | **SQL-инъекции** | ✅ ОК | Используется только Prisma ORM, сырых SQL-запросов с подстановкой строк нет. |
| 2 | **XSS (рендеринг пользовательского текста как HTML)** | ✅ ОК | В `script.js` данные заявок перед вставкой в `innerHTML` проходят через `escapeHtml()`. В `admin/app.js` также используется `escapeHtml()` для данных из API. |
| 3 | **CORS** | ✅ Исправлено | В `main.ts` используется `CORS_ORIGIN` из .env (список доменов через запятую). Если не задан — разрешён любой источник (удобно для разработки). Для продакшена задайте `CORS_ORIGIN=https://yourdomain.com`. |
| 4 | **Rate limit на логин/регистрацию** | ✅ Исправлено | Подключён `@nestjs/throttler`: на `POST /auth/login` и `POST /leads` действует лимит 10 запросов в минуту и 30 за 5 минут. |
| 5 | **Stack trace в ответах (продакшен)** | ✅ ОК | В коде stack используется только в `logger.error()`, в HTTP-ответы не попадает. NestJS по умолчанию не отдаёт stack в теле ответа. Рекомендуется задать `NODE_ENV=production` при деплое. |
| 6 | **Захардкоженные секреты** | ✅ Частично исправлено | В продакшене (`NODE_ENV=production`) приложение не запустится без заданного `JWT_SECRET`. Рекомендуется также задать `CURSOR_SECRET` в .env. Мок `admin/server.js` и seed — только для разработки; в проде не использовать без смены пароля. |
| 7 | **Загрузка файлов: размер и тип** | ✅ ОК | В `PresignedUrlDto`: размер ограничен `@Min(1)` и `@Max(MAX_UPLOAD_SIZE)` (10 MB), имя файла и `mimeType` ограничены по длине. В `SaveFileMetadataDto`: размер и MIME проверяются. Для ужесточения можно добавить allowlist допустимых MIME-типов. |

---

## Рекомендуемые исправления

### 3. CORS
Ограничить `origin` списком разрешённых доменов (и при необходимости портов для разработки).

### 4. Rate limit
Подключить `@nestjs/throttler` и применить лимиты к `AuthController` и к `LeadsController` (POST).

### 6. Секреты
- В продакшене обязательно задавать `JWT_SECRET` и `CURSOR_SECRET` в окружении (не полагаться на значения по умолчанию).
- Админку в проде обслуживать только NestJS, не использовать `admin/server.js` с захардкоженным паролем.
